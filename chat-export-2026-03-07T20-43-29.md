The Protocall folder contains the "Fair Hiring Interview" module—a sophisticated, AI-driven interview simulation tool. Its primary role in the main application is to provide candidates with a realistic practice environment and generate deep-dive intelligence reports on their performance.

Here is a detailed breakdown of what it does and how it works:

1. High-Level Role
In the wider project, Protocall is the "Candidate Experience" bridge. While other parts of the app handle hiring pipelines for companies, Protocall acts as a training and evaluation ground for applicants, accessible via the /candidate/interview route.

2. How It Works (The Lifecycle)
The system operates in three distinct phases managed by 

ProtocallApp.jsx
:

Phase A: Configuration (

InterviewSetup.jsx
)
Before the interview begins, the candidate configures the AI agent. They can specify:

Target Role: (e.g., Senior Software Engineer)
Company Context: (e.g., Google, OpenAI)
Evaluation Focus: They can select specific traits like "System Design," "Communication," or "Problem Solving" for the AI to prioritize.
Phase B: The Real-Time Session (

InterviewSession.jsx
)
This is the "engine room" of the folder. It uses the Gemini Live API to create a multimodal experience:

Multimodal Input: It streams the user's microphone (audio) and camera (video frames) directly to Gemini.
Real-Time Observation: Because it "sees" and "hears" the user, the AI (named "Puck") can give immediate feedback on visual cues, such as posture or eye contact, using a custom-defined tool called updateVisualFeedback.
AI Personality: The AI is instructed to act as an elite executive interviewer, asking probing follow-up questions rather than just static ones.
Phase C: The Intelligence Report (

AnalysisReport.jsx
)
Once finished, the 

analysisService.js
 sends the full transcript to Gemini for a structured evaluation. It returns:

Numerical Scores: On Clarity, Confidence, and Technical Knowledge.
Visual Charts: Uses recharts to create a "Radar Map" and "Skill Segmentation" bars.
Strengths & Growth: Provides actionable pointers and a searchable, line-by-line transcript of the session.
3. Key Technical Components

ProtocallApp.jsx
: The main orchestrator (State machine).

InterviewSession.jsx
: Handles the complex WebRTC-like streaming logic and Gemini Live connection.

analysisService.js
: A specialized service that prompts Gemini to return structured JSON data for the charts.

audioUtils.js
: Helper functions to handle PCM audio conversion for the AI's "ears."

constants.jsx
: Stores the design system icons and the specific Gemini model versions used.
4. Aesthetics & Design
The folder follows the project's "Neo-Grotesque" design language—high contrast, heavy typography (Montreal and Grotesk fonts), and "brutalist" UI elements like thick borders and offset shadows.
