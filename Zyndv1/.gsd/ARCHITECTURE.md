## Overview

The **Fair Hiring Platform** is a multi-agent AI system designed to eliminate bias and fraud in recruitment. It consists of three main layers: a React frontend, a FastAPI backend orchestrator, and 10 independent microservice agents that verify candidate skills across multiple platforms (GitHub, LeetCode, Codeforces, LinkedIn).

1. ATS fraud detection вҶ’ 2-5. Parallel scraping (GitHub, LeetCode, Codeforces, LinkedIn) вҶ’ 6. Skill aggregation вҶ’ 7. Conditional testing вҶ’ 8. Bias detection вҶ’ 9. Job matching вҶ’ 10. Cryptographic passport generation

## Data Flow

### Application Submission Flow
1. **Candidate applies** вҶ’ Frontend sends resume + profile URLs to backend
2. **Backend creates** application record in PostgreSQL
3. **Pipeline orchestrator** triggers 10-stage workflow
4. **Stage 1: ATS Agent** analyzes resume for fraud
   - PASS вҶ’ Continue to Stage 2
   - NEEDS_REVIEW вҶ’ Queue for human review
   - BLACKLIST вҶ’ Reject immediately
5. **Stages 2-5: Parallel scraping** (GitHub, LeetCode, Codeforces, LinkedIn)
   - Each agent fetches and analyzes platform data
   - Results stored in `credentials` table
6. **Stage 6: Skill Agent** aggregates evidence
   - Combines signals from all sources
   - Calculates confidence scores per skill
   - Determines if additional testing needed
7. **Stage 7: Conditional Test** (if confidence < threshold)
   - Generates skill-specific assessment
   - Pipeline pauses until test completion
8. **Stage 8: Bias Detection** analyzes hiring patterns
   - Checks for discrimination signals
   - Flags systemic issues
9. **Stage 9: Matching Agent** compares to job requirements
   - Calculates match score
   - Identifies skill gaps
10. **Stage 10: Passport Agent** generates credential
    - Creates JSON with verified skills
    - Signs with Ed25519 private key
    - Stores in `credentials` table

### Authentication Flow
- JWT-like tokens issued on signup/login
- Tokens stored in frontend localStorage
- Sent in `Authorization: Bearer <token>` header
- Backend validates and extracts user_id/role

### Database Access Pattern
- Async SQLAlchemy with asyncpg driver
- Connection pooling for performance
- Alembic for schema migrations
- JSONB columns for flexible data (requirements, feedback, credentials)

**Environment Variables:**
- Centralized in `.env` files
- Separate configs for backend, agents, frontend
- Sensitive keys (GITHUB_PAT, OPENAI_API_KEY) in .env

## Security Architecture

**ATS Guard System** (Multi-layer fraud detection):
1. **Structural Layer** - White text detection, tiny font detection
2. **Prompt Injection Layer** - Hard/soft injection detection
3. **Semantic Layer** - Manipulation pattern analysis
4. **Response Schema** - Graduated trust scoring (OK, NEEDS_REVIEW, BLACKLIST)

**Cryptographic Signing:**
- Ed25519 key pair for skill passports
- Private key (SIGNING_PRIVATE_KEY_B64) stored in backend .env
- Public key (SIGNING_PUBLIC_KEY_B64) for verification
- SHA-256 hash of credential JSON before signing

**Authentication:**
- Password hashing with bcrypt
- JWT-like tokens with python-jose
- Role-based access control (candidate, company, admin)
