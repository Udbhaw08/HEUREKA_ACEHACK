## 🚨 Important design correction (very important)

> ❌ “this agent id should be generated by the agent”

**NO.**
The agent should **not** generate it.

### ✅ Correct owner of `evaluation_id`

👉 **Pipeline Orchestrator / Backend API**

Reason:

* Agents must be **stateless**
* IDs must be traceable
* DB transactions must be atomic
* Frontend ↔ backend must agree on ID

### Step 1️⃣ User logs in

**Input**

### Step 4️⃣ Conditional Test Agent

Updates same `evaluation_id`

### Step 5️⃣ Bias Detection

Reads `evaluation_id` + private metadata

### Step 6️⃣ Matching Agent

Ranks **this evaluation** against job pool
